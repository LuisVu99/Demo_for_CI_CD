# Automation Exercise Demo Project

This repository contains an automation test framework built with Python, Playwright and Pytest. It is intended for UI and API testing and integrates with Allure for reporting. This README provides platform-friendly (PowerShell) instructions for setup, running tests, and generating reports.

## Quick facts

- Project name: Automation Exercise Demo Project
- Language: Python
- Test runner: Pytest
- Automation: Playwright (UI & API)
- Reporting: Allure

---

## Requirements

- Python 3.10+ (or compatible)
- pip
- Playwright Python package and its browsers (Chromium/Firefox/WebKit)
- (Optional) Allure Commandline to generate local HTML reports

---

## Setup (Windows PowerShell)

1) Create and activate a virtual environment (recommended):

```powershell
python -m venv .venv; .\.venv\Scripts\Activate.ps1
```

2) Install project dependencies:

```powershell
pip install -r requirements.txt
```

3) Install Playwright browser binaries:

```powershell
playwright install
```

Note: if Playwright is not listed in `requirements.txt`, run `pip install playwright` first.

If you plan to use Allure locally, install the Allure CLI on your machine (via Chocolatey, Scoop, or manual download) or use a container/CI to generate reports.

---

## Running tests

Examples (PowerShell):

- Run tests with default browser (example uses Chromium):

```powershell
pytest --env=dev --browser=chromium
```

- Run tests across all browsers (if supported by the framework):

```powershell
pytest --env=dev --browser=all
```

- Run tests in parallel (requires `pytest-xdist`):

```powershell
pytest -n 3 --env=dev --browser=chromium
```

- Retry flaky tests (requires `pytest-rerunfailures`):

```powershell
pytest --env=dev --browser=chromium --reruns 2
```

Notes:
- The `--env` and `--browser` options depend on your `conftest.py`/`pytest.ini` implementation. Adjust or remove flags if not available.
- Use `pytest -q` for more concise output or `-k <expr>` to run a subset of tests.

## Allure reporting (local)

After tests run, pytest should create results in `allure-results/` (or a configured results folder).

- Generate and open a local Allure report (PowerShell):

```powershell
allure generate .\allure-results --clean -o .\allure-report; allure open .\allure-report
```

- Or serve the results directly:

```powershell
allure serve .\allure-results
```

If Allure CLI is not installed, consider using GitHub Actions to publish the report or install Allure via your package manager.

## Troubleshooting

- Missing packages: ensure virtual environment is active and run `pip install -r requirements.txt`.
- Playwright browser errors: run `playwright install` to download required browsers.
- Allure not found: install Allure CLI or generate reports in CI where Allure is available.
